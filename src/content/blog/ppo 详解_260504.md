## 什么是 PPO

PPO 的全称是 ***Proximal Policy Optimization***，中文叫做近端策略优化。

## PPO 需要的模型

- **Policy model**：最终训练的模型。
- **Reference model**：用于 KL 散度约束，确保模型不偏离原本的模型太远。
- **Reward model**：用于判断模型输出结果是否正确，并给出对应的 reward 打分。
- **Old policy model**：旧的 policy model，用来和当前 policy model 对比打分。
- **Value model**：又叫 critic model，用来给回答的 token 逐 token 打分。

## 具体运行过程

1. 输入 prompt 给 old policy model，获得对应的 response 和 logprobs。
2. 根据输入的 prompt 和给出的 response，输入到 reward model 中，得到对应的 reward。
3. 将 prompt 和 response 输入到 value model 中，对 response 逐 token 获得其 value。
4. 通过 GAE 或朴素算法 `reward - value`，得到 response 的每个 token 对应的 advantage。
5. 输入 prompt 与 response，获得相同输出结果对应的 `new_logprobs`。计算 `new_logprobs - logprobs`，再计算对应自然指数 `e`，得到最后的 ratio。
6. 计算 clipped policy loss。

PPO clipped policy loss 的公式是：

$$
L_t^{PPO}
=
\min\left(
\rho_t A_t,
\operatorname{clip}(\rho_t, 1-\epsilon, 1+\epsilon) A_t
\right)
$$

在这里，$\rho_t$ 是第五步计算出的 ratio，$A_t$ 是第四步计算出来的 advantage。

这一步为了防止 $\rho_t$ 的值过大或过小，会设置一个 $\epsilon$ 来进行抑制。

比如这里的 $\epsilon$ 设置为 0.2，那么范围就是 $[0.8, 1.2]$。当 $\rho_t$ 的值超出这个范围后，会被强制裁剪回这个范围中。例如，当 $\rho_t=0.7$ 时会裁剪到 0.8；当 $\rho_t=1.4$ 时会裁剪到 1.2。然后用裁剪后的值和 $A_t$ 相乘，得到另一个结果。最后取两个结果的最小值作为 $L_t^{PPO}$ 的值。

## Value Model 和 Returns

PPO 学习过程中，除了要学习新的 policy model 外，还要学习 value model。

这里的 value model 需要一个新的计算参数，叫做 returns，其含义是：

**从当前位置开始，未来能获得的累计奖励。**

一般形式是：

$$
G_t
=
r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots
$$

这里，$G_t$ 表示第 $t$ 步的 return；$r_t$ 表示第 $t$ 步的即时 reward。在平常的计算中，一般默认只有最后一步得到结果的时候才有 reward，而其他步的 reward 一般为 0；$\gamma$ 的含义是折扣因子。

这里我们假设没有 KL 散度，所以直接遵循上面的公式即可。实际计算时一般是逆推，从最后一个 token 逆推到第一个 token。

假设条件为：

$$
\gamma = 0.9,
\quad
r = [0, 0, 0, 0, 1]
$$

则有：

$$
\begin{aligned}
G_5 &= 1 \\
G_4 &= 0 + 0.9 \times 1 = 0.9 \\
G_3 &= 0 + 0.9 \times 0.9 = 0.81 \\
G_2 &= 0 + 0.9 \times 0.81 = 0.729 \\
G_1 &= 0 + 0.9 \times 0.729 = 0.6561
\end{aligned}
$$

基于这里的 returns，可以根据 value model 原本输出的 value 数值计算 MSE，得到最后的 value loss。

$$
\mathcal{L}_{value}
=
\frac{1}{5}
\sum_{t=1}^5
(V_t-G_t)^2
$$

## KL Loss

这一步会通过 reference model 计算对应的 `ref_logprobs`。

传统 KL loss 会计算每一个 token 的概率，最后将结果求和。这里的简化版本只看当前 old policy model 输出的指定 response 中 token 的概率，并计算 reference model 和 policy model 之间的差距。

详见：

[sampled KL vs strict KL](https://www.notion.so/sampled-KL-vs-strict-KL-356abf2572ee803fbea5ecba56725713?pvs=21)

最终，这里计算 KL loss 的公式如下：

$$
\mathcal{L}_{KL,t}
=
\log \pi_\theta(y_t \mid s_t)
-
\log \pi_{\text{ref}}(y_t \mid s_t)
$$

这里 KL 除了本身对 loss 有影响之外，还会对上面的 value loss 产生相关影响。

具体来说，影响产生在 returns 的计算过程中。

这里有：

$$
r_t^{total}
=
r_t^{task}
-
\beta \mathrm{KL}_t
$$

举例来说，假设这里有：

$$
r = [0, 0, 0, 0, 1]
$$

$$
\mathrm{KL} = [0.02, 0.01, 0.03, 0.01, 0.04]
$$

$$
\beta = 0.1
$$

那么根据实际情况可以得到：

$$
r_t^{total}
=
r_t^{task}
-
0.1 \mathrm{KL}_t
$$

经过计算得到：

$$
r^{total}
=
[-0.002, -0.001, -0.003, -0.001, 0.996]
$$

假设 $\gamma = 1$，可以得到：

$$
\begin{aligned}
G_5 &= 0.996 \\
G_4 &= -0.001 + 0.996 = 0.995 \\
G_3 &= -0.003 + 0.995 = 0.992 \\
G_2 &= -0.001 + 0.992 = 0.991 \\
G_1 &= -0.002 + 0.991 = 0.989
\end{aligned}
$$

这也就是最后的 return：

$$
\mathrm{returns}
=
[0.989, 0.991, 0.992, 0.995, 0.996]
$$

## 总 Loss

根据上面的 loss 计算，可以得到 PPO 训练的最终 loss 函数：

$$
\mathcal{L}
=
L_t^{PPO}
+
c_v \mathcal{L}_{value}
+
\beta \mathcal{L}_{KL}
$$

## 反向传播

伪代码过程如下，这里标号和上面的数字标号无关：

```python
for step in range(num_steps):
    prompts = sample_prompts()

    # 1. old policy 采样回答
    responses, old_logprobs = old_policy.generate(prompts)

    # 2. reward model / rule 打分
    rewards = reward_fn(prompts, responses)

    # 3. value model 估计每个 token 的 value
    values = value_model(prompts, responses)

    # 4. 计算 returns / advantages
    returns, advantages = compute_gae_or_returns(rewards, values)

    # 5. 当前 policy 计算 new logprobs
    new_logprobs = policy_model.logprobs(prompts, responses)

    # 6. 计算 ratio
    ratios = exp(new_logprobs - old_logprobs)

    # 7. PPO clipped loss
    policy_loss = clipped_ppo_loss(ratios, advantages)

    # 8. value loss
    value_loss = mse(values, returns)

    # 9. KL penalty
    ref_logprobs = ref_model.logprobs(prompts, responses)
    kl_loss = compute_kl(new_logprobs, ref_logprobs)

    # 10. 总 loss
    loss = policy_loss + c_v * value_loss + beta * kl_loss

    # 11. 更新
    loss.backward()
    optimizer.step()
```

除此之外，loss 还可以有 entropy loss，用来鼓励分布更平滑一些。

但是 entropy bonus 并不是所有实现都显式使用，所以这里可以自行了解，不再强调。
