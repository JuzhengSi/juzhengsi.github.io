PPO 是什么

PPO 的全称是 Proximal Policy Optimization，中文叫近端策略优化。它的核心是：更新 policy model 时，让模型朝着更高 reward 的方向移动，但不要一步偏离旧策略太远。

这篇笔记把 PPO 训练拆成几个可观察对象：先采样回答和 logprobs，再计算 reward、value、advantage、ratio，最后把 clipped policy loss、value loss 和 KL loss 合成总目标。

PPO 需要的模型

Policy model最终要训练和更新的模型。
Reference model提供 KL 约束，避免 policy 偏离初始模型太远。
Reward model判断回答质量，并给出 reward 打分。
Old policy保留旧策略的 logprobs，用来和当前 policy 计算 ratio。
Value model也叫 critic model，为 response token 逐个估计 value。
Optimizer把所有 loss 汇总后反向传播更新参数。

具体运行过程

01输入 prompt 给 old policy，得到 response 和 old logprobs。
02把 prompt 和 response 输入 reward model，得到 task reward。
03用 value model 对 response 的每个 token 估计 value。
04根据 reward 和 value 计算 returns / advantages。
05当前 policy 重新计算相同 response 的 new logprobs。
06计算 ratio，再进入 clipped policy loss。

Clipped Policy Loss

ratio 来自当前 policy 和 old policy 的概率差异。如果 ratio 太大或太小，PPO 会把它裁剪到一个安全范围内，避免单次更新过猛。

$$ L_t^{PPO} = \min\left( \rho_t A_t, \operatorname{clip}(\rho_t, 1-\epsilon, 1+\epsilon) A_t \right) $$
epsilon = 0.2clip range = [0.8, 1.2]
ratio 越界0.7 -> 0.8，1.4 -> 1.2

这里的重点不是简单取最小值，而是用裁剪后的目标限制策略变化，让训练更稳定。

Value Model、Returns 和 KL

Value model 学习的是从当前位置开始未来能获得的累计奖励。常见计算方式是从最后一个 token 逆推到第一个 token。

$$ G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots $$

加入 KL 后，reward 会被 reference model 和 policy model 的差距惩罚修正。

$$ r_t^{total} = r_t^{task} - \beta \mathrm{KL}_t $$ $$ \mathrm{KL}_t = \log \pi_\theta(y_t \mid s_t) - \log \pi_{\mathrm{ref}}(y_t \mid s_t) $$

因此 KL 不只影响 loss 项，也会进入 returns 的计算过程，改变 value model 的学习目标。

总 Loss 与反向传播

最终训练目标把 policy 的 clipped loss、value model 的 MSE 以及 KL penalty 合在一起。

$$ \mathcal{L} = L_t^{PPO} + c_v \mathcal{L}_{value} + \beta \mathcal{L}_{KL} $$

训练循环可以概括为：old policy 采样、reward 打分、value 估计、计算 returns/advantages、当前 policy 计算 new logprobs、ratio 进入 clipped loss，最后汇总 loss 并反向传播。
